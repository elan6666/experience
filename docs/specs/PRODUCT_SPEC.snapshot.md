# Product Specification — Seven-Model A-Share Research

## Positioning

面向个人研究员的小团队式A股模型与策略实验平台。它不是“挑一个最高收益模型”，而是把PIT数据、论文模型、共同评估、风险预算、组合约束和失败诊断组织成一条可逐层证伪的流水线。

## Target users

- Primary：项目所有者/量化研究员。
- Secondary：复核数据、代码、统计和交易可执行性的研究协作者。

## Problems and jobs to be done

1. 在不修改论文训练目标的前提下，公平比较不同类型模型。
2. 判断PIT因子与统一市场状态是否真正提供增量。
3. 判断收益改善来自选股、择时、成本控制还是模型融合。
4. 在结果差时区分数据、实现、适配、训练不足和模型本身失效。
5. 生成可重放的研究证据，而不是不可审计的单次回测。

## MVP scope

### D0 — 数据与协议

- 四股票池、历史点时成员、PIT因子、市场状态和A股可交易状态。
- 训练集拟合的填补/标准化/中性化与逐因子缺失标记。
- 信号时点、T+1、标签、成本、停牌/涨跌停和覆盖率协议。

### V0 — 七模型基础比较

- 七模型 × 四股票池的Core输入比较。
- 官方代码/版本、适配边界、3-seed稳定性和失败分类。
- 与等权、动量、现金和官方指数比较。

### V1 — 七模型输入消融

- A0 Core、A1 Core+F、A2 Core+S、A3 Core+F+S。
- 周频5日标签作为主选择协议，完整scorecard共112个模型—股票池—输入配置；A0直接复用V0的28个结果，V1只新增84个训练配置。
- 日频、月频、Top-K和因子组删除作为冻结后的敏感性实验。

### V2 — 策略层

- B0始终满仓、B1统一100/60/30/0风险预算、B2受约束组合、B3条件式融合。
- 只有通过验证门槛且具有互补性的模型才进入融合。

## Requirements

- 七模型必须在四股票池分别训练、选择和报告；不得用跨池平均掩盖单池失败。
- 沪深300和科创50使用历史点时成分；科技32/100标记为2026名单条件实验。
- 成员、观察、可交易和逐因子缺失掩码属于合法性基础设施，不能作为可关闭的性能增强。
- 论文模型保留官方架构、loss、优化器、调度器和推理逻辑；适配仅位于外部数据/导出层并留痕。
- 所有候选和信号基线使用同一标签、信息截止、T+1、成本、风险预算与组合规则。
- 主预测门槛使用未来5日横截面RankIC；主经济结果使用相对相应官方指数或同池可交易等权的5日可执行超额收益，同时辅助报告绝对收益。论文模型的原生训练loss/target保持不变。
- 同时报告共同可评分支持集和模型原生覆盖率结果。
- 测试数据不得用于早停、参数、Top-K、因子、融合权重或风险阈值选择。

## Non-goals

- 不连接实盘券商，不自动下单，不承诺收益。
- 不把A股适配称为原论文数据集上的精确复现。
- 不修改训练loss来迎合统一评估。
- 第一阶段不加入新闻、LLM或高频订单簿。
- 不把科技32/100回溯结果称为无幸存者偏差的全市场alpha。
- 停放在FUTURE.md的Git发布任务不属于当前MVP。

## Acceptance criteria

1. 每个股票池有数据清单、哈希、PIT/成员/覆盖率审计和明确状态。
2. 每个模型有上游版本、许可证/来源、训练命令、配置、随机种子和适配差异。
3. V0 28个主单元均为完成、明确失败或因数据门槛阻断，不允许静默缺失。
4. V1完整112格scorecard按冻结协议完成或留下可复核阻断原因，且A0哈希与V0完全一致。
5. 每张模型表同时包含预测、经济、可交易、覆盖率和统计稳定性指标。
6. V2逐层报告B0/B1/B2/B3增量，不用最终比较集反向挑规则。
7. 2026-01至2026-07-17明确标记为已查看研究比较集；最终结论等待冻结后的新数据。
8. 交付时能重放核心结果，并由独立复核确认代码问题与模型弱表现的归因。
